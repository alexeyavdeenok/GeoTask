# 🔒 Слой безопасности GeoTask

## Описание

Добавлен слой шифрования между DAO и БД для защиты чувствительных данных.

### Что шифруется:

**Tasks:**
- `title` - название задачи
- `description` - описание задачи

**Locations:**
- `name` - имя места
- Координаты (широта/долгота)

**Settings:**
- Все поля с пользовательскими настройками

## Как это работает

1. **Перед сохранением в БД:**
   - Data Layer → `SecureTaskDao/SecureLocationDao/SecureSettingsDao`
   - Здесь данные шифруются с помощью AES-256-GCM
   - Зашифрованные данные отправляются в обычный DAO
   - DAO сохраняет их в БД

2. **При чтении из БД:**
   - Обычный DAO читает зашифрованные данные
   - `SecureTaskDao/SecureLocationDao/SecureSettingsDao` расшифровывают данные
   - Расшифрованные данные возвращаются в Repository/ViewModel

## Архитектура

```
Repository ↓↑ SecureDAO ↓↑ RegularDAO ↓↑ Database
             (шифрует/    (обычные
              расшифрует) операции)
```

## Технические детали

- **Алгоритм:** AES-256 с режимом GCM
- **Хранение ключей:** Android KeyStore (защищено системой)
- **IV:** 12 байт, генерируется для каждого шифрования
- **Кодирование:** Base64 для хранения бинарных данных

## Обработка ошибок

Если произойдёт ошибка при расшифровке (например, ключ изменился), система возвращает оригинальные данные без расшифровки. Это позволяет избежать краша приложения.

## Миграция данных

⚠️ **Важно:** При первом запуске приложения с этой версией:
- Старые нешифрованные данные останутся в БД
- Новые данные будут сохраняться в зашифрованном виде
- При чтении старых данных система попытается расшифровать, но вернёт оригинальные значения

Для полного шифрования всех данных можно добавить миграцию БД.

## Использование в коде

Вы не должны ничего менять! Слой безопасности внедрён через Dependency Injection:

```kotlin
// В вашем коде просто используйте DAO как раньше
repository.insertTask(myTask) // Автоматически шифруется
```

## Отключение шифрования (для отладки)

Если нужно временно отключить шифрование, в `AppModule.kt` можно заменить:

```kotlin
// Вместо:
@Provides
@Singleton
fun provideTaskDao(
    db: AppDatabase,
    encryptionManager: EncryptionManager
): TaskDao = SecureTaskDao(db.taskDao(), encryptionManager)

// На:
@Provides
@Singleton
fun provideTaskDao(db: AppDatabase): TaskDao = db.taskDao()
```

## Производительность

- Шифрование/расшифровка добавляет ~1-5мс на операцию (зависит от размера данных)
- Используется AES-256-GCM, который аппаратно ускорен на современных процессорах
- Ключ хранится в памяти защищённо и создаётся один раз при запуске приложения

## Безопасность

- ✅ Данные зашифрованы на диске
- ✅ Ключ хранится в защищённом хранилище Android
- ✅ Используется криптографически стойкий алгоритм AES-256
- ✅ IV (инициализирующий вектор) генерируется случайно для каждого шифрования
- ✅ Аутентификация гарантирует целостность данных (GCM режим)

## Файлы

- `EncryptionManager.kt` - основной менеджер для шифрования/расшифровки
- `SecurityKeyStore.kt` - управление криптографическими ключами
- `SecureTaskDao.kt` - защищённый DAO для Tasks
- `SecureLocationDao.kt` - защищённый DAO для Locations
- `SecureSettingsDao.kt` - защищённый DAO для Settings
