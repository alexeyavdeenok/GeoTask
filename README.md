## Build APK

1. Clone the repo:

```bash
git clone https://github.com/alexeyavdeenok/GeoTask.git
cd geotask
```

2. Run build:

Linux / Mac:

```bash
./gradlew assembleDebug
```

Windows:

```bash
gradlew assembleDebug
```

3. APK will be here:

```
app/build/outputs/apk/debug/app-debug.apk
```

### Руководство пользователя

Интерактивная карта: В приложении доступна карта с метками ваших локаций. Для их корректного отображения необходимо разрешить доступ к местоположению при первом запуске.

Актуальная погода (API): На главном экране автоматически выводятся данные о температуре и погодных условиях. Информация подгружается в реальном времени через открытый API Open-Meteo.

Управление местами: Список всех сохраненных точек с названиями доступен в разделе «Места» для быстрого просмотра ваших геозон.

## Безопасность и конфиденциальность

### Шифрование данных

Все чувствительные данные (названия задач, описания, координаты и настройки) шифруются перед сохранением в локальную базу данных. Используется алгоритм AES-256 в режиме GCM. Это гарантирует, что при физическом доступе к файлу базы данных (.db) содержимое останется нечитаемым.

### Хранение ключей

Криптографические ключи генерируются и хранятся внутри Android KeyStore. В зависимости от возможностей устройства, ключи защищаются на аппаратном уровне (через TEE или StrongBox), что исключает их извлечение программным путем.

### Принципы обработки данных

**Локальное хранение:** Основной массив данных остается на устройстве пользователя.

**Минимизация запросов:** Приложение обращается к внешнему API (Open-Meteo) только для получения метеоданных. При этом идентификационные данные пользователя или история перемещений на сервер не передаются.

**Разрешения:** Запрашиваются только доступы к геолокации (ACCESS_FINE_LOCATION, ACCESS_COARSE_LOCATION), необходимые для работы основной логики геозон.

## Монетизация проекта

Стратегия монетизации GeoTask строится на поиске баланса между бесплатными возможностями и окупаемостью разработки. Так как приложение ориентировано на рынок РФ, все инструменты адаптированы под локальные сервисы.

### Как приложение может зарабатывать

Планируется использовать гибридную модель: приложение остается бесплатным, но с рекламой и возможностью купить Premium-функции.

#### 1. Реклама (Yandex Ads)
- **Как устроено**: Интеграция мобильной рекламы от Яндекса.
- **Формат**: Небольшой баннер внизу экрана, который не мешает пользоваться картой и списком задач.

#### 2. Платные функции (RuStore)
Для приема платежей в рублях (через карты РФ и СБП) подключается SDK магазина RuStore. Пользователям предлагается два варианта:

- **Бесплатно**: Создание любых задач и геозон, просмотр погоды, стандартные уведомления.
- **Premium (Подписка или разовая покупка)**:
  - Полное отключение рекламы.
  - Экспорт истории задач и мест в файлы CSV или PDF.
  - Дополнительные настройки (например, выбор уникальных звуков для уведомлений).

#### 3. Донаты
- **Как устроено**: Простая кнопка «Поддержать автора» в настройках приложения, ведущая на форму чаевых (например, CloudTips). Подходит для тех, кто хочет помочь проекту добровольно.

### Перспективы развития

Сейчас проект полностью автономен и работает локально на устройстве. Если в будущем возникнет необходимость выйти на зарубежный рынок, архитектура приложения позволит легко это сделать. Потребуется лишь собрать отдельную версию (Flavor) в Gradle, заменив сервисы Яндекса и RuStore на AdMob и Google Play Billing.
